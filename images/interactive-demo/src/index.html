  1. One MCP entrypoint. The LLM only sees POST /mcp and a tidy tools list — protocol heterogeneity is invisible.
  2. Three protocols, in parallel. KrakenD fans out to REST (entity), GraphQL (ownership) and SOAP (sanctions) concurrently, decoding XML and querying GraphQL natively.

  4. Output reshaped for the consumer. KrakenD's response-transformation layer converts the merged JSON into compact TOON text — fewer tokens, identical information. A second tool serves verbose JSON for comparison.
