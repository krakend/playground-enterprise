Generate & observe LLM traffic

Fire real calls across every wired provider so the dashboards and traces have something to show. Use the button for a quick one-click burst, or the script for sustained, configurable load in a terminal — then open the surfaces below to watch token usage, latency, status and end-to-end traces.

# Sustained traffic from a terminal (runs 60s and stops by default)
./scripts/demo-llm-traffic.sh

# Longer / heavier runs
DURATION=120 ./scripts/demo-llm-traffic.sh
ROUNDS_BURST=3 DELAY=1 ./scripts/demo-llm-traffic.sh
  • Grafana · AI GatewayToken usage per provider and model (in / out / total).
  • Grafana · OpenTelemetry + PrometheusThroughput, latency percentiles, status codes per endpoint and backend.
  • JaegerPer-request traces: latency, backend/model, response status. Service “KrakenD”.
  • PrometheusRaw metrics, e.g. krakend_llm_tokens_total by provider / model.
