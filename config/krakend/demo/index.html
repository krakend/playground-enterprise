Playground

This is a demonstration environment that puts together the necessary pieces to get you started with our API Gateway, using example use-cases.

Explore demos for these use cases:

    • Aggregation and basic transformation
    • Sequential calls, using values from 1st call response into 2nd call request
    • Advanced transformation with query language
    • Regular expression manipulation
    • Mocked response
    • Security policy engine for requests, responses and claim checking
    • Two types of rate limiting on aggregated content
    • Backend cache
    • Concurrent calls - Initial example with no concurrent calls enabled (normal)
    • Concurrent calls - Example with concurrent calls enabled (more speed, less errors)
    • GraphQL to REST
    • Expose a GET REST endpoint from POST to SOAP service
    • GRPC
    • GRPC Payload in Body
    • WebSockets (chat example)
    • WebSockets (chat example web interface)
    • Web Server - Static content
    • Wildcard to our fake API, open
    • Wildcard to our fake API, protected with Basic Auth
    • Wildcard with geofencing - Allow visits only from Spain
    • Dynamic routing based on headers (token claim and cookie also available)
    • Dynamic routing based on query strings
    • Only POST method allowed
    • A public endpoint that requires no authentication
    • Authorization based on JWT (using Auth0)
    • Authorization based on JWT (using KrakenD JOSE signer)
    • JWT signing, as described at https
    • Authorization based on API Keys
