To run this demo end-to-end you need two things in place:

1. License

AI Gateway is an Enterprise add-on. Standard EE licenses don't include it — if your license doesn't enable AI Gateway, the endpoint will reject the request at boot. Reach out to your account contact at KrakenD to enable it on your license.

2. Provider credentials

Add your GEMINI_API_KEY, OPENAI_API_KEY and ANTHROPIC_API_KEY to config/krakend/.env.local, then start the playground with:

make start-with-ai-gateway

Without keys the endpoint still responds, but the upstream LLM call will return an error.

Multi-Provider LLM Routing

Route AI requests to three different LLM providers from a single endpoint — no application code changes needed. KrakenD inspects the X-Model header and forwards the request to the right provider.

How it works

The gateway uses conditional backends with a header-based strategy. Each backend targets a different LLM provider and activates only when its condition matches:

"backend/conditional": {
  "name": "X-Model",
  "strategy": "header",
  "value": "gemini"
}

When no header matches, a fallback backend handles the request — in this case, Anthropic.

Supported Providers

  • X-Model: gemini → Google Gemini (gemini-2.5-flash)
  • X-Model: openai → OpenAI (gpt-4o-mini)
  • No header → Anthropic fallback (claude-3-haiku)

Try it

Try it interactively in the Interactive Demos, or via curl:

# Route to Gemini
curl -XPOST -H'X-Model: gemini' \
  --json '{"contents":"What is an API gateway?"}' \
  http://localhost:8080/llm-routing

# Route to OpenAI
curl -XPOST -H'X-Model: openai' \
  --json '{"contents":"Write a haiku about APIs"}' \
  http://localhost:8080/llm-routing

# Default to Anthropic (no header)
curl -XPOST \
  --json '{"contents":"Name 3 benefits of API gateways"}' \
  http://localhost:8080/llm-routing

Why this matters

Switch LLM providers, run A/B tests between models, or set up cost-optimized fallback chains — all from gateway configuration, without touching your application.

Note: This demo uses the X-Model header as the routing key, but KrakenD's conditional routing supports any strategy: headers, JWT claims, query parameters, cookies, or custom CEL policies. You define the rules that fit your architecture.
